代码对比示例
① 类型安全 最核心的区别
JavaScript ❌
function add(a, b) {
  return a + b;
}

add(1, "2");
// 返回 "12"（字符串拼接）
// 运行时才发现，无任何警告
TypeScript ✅
function add(
  a: number,
  b: number
): number {
  return a + b;
}

add(1, "2");
// 🔴 编译错误！编写时就报警
JS 中 1 + "2" 会进行字符串拼接返回 "12"， TS 在你写代码的瞬间就提示你传了错误类型，不等程序运行。
② 接口 Interface TS 独有特性
JavaScript
// JS 没有接口，只能靠注释约定
// @param {Object} user
// @param {number} user.id
// @param {string} user.name
function greet(user) {
  return `你好, ${user.name}`;
}

// 传错了也不报错
greet({ id: 1, nome: "李天宇" });
// 返回 "你好, undefined" 😱
TypeScript
interface User {
  id: number;
  name: string;
  age?: number; // ? 可选字段
}

function greet(user: User) {
  return `你好, ${user.name}`;
}

greet({ id: 1, nome: "李天宇" });
// 🔴 错误：没有 name，多了 nome
Interface 相当于一份合同，明确规定对象必须有哪些字段， 字段类型是什么，哪些是可选的。传错了立刻报警。
③ 泛型 Generic TS 独有特性
JavaScript
// 写通用函数，类型信息丢失
function getFirst(arr) {
  return arr[0];
}

const n = getFirst([1, 2, 3]);
// n 的类型？不知道...
// IDE 无法给出精准提示

const s = getFirst(["a", "b"]);
// s 的类型？还是不知道...
TypeScript
// T 是占位符，调用时自动推断
function getFirst<T>(arr: T[]): T {
  return arr[0];
}

const n = getFirst([1, 2, 3]);
// ✅ n 的类型是 number

const s = getFirst(["a", "b"]);
// ✅ s 的类型是 string
泛型让函数既通用又类型安全。 <T> 是一个「类型占位符」，调用时自动推断出实际类型， IDE 的补全提示也更精准。
④ 枚举 Enum TS 独有特性
JavaScript
// 只能用常量模拟，易写错
const STATUS = {
  PENDING: "pending",
  SUCCESS: "success",
  ERROR:   "error"
};

function handle(s) {
  if (s === "succes") { // 拼写错了！
    // 不会报错，静默 bug
  }
}
TypeScript
enum Status {
  Pending = "pending",
  Success = "success",
  Error   = "error"
}

function handle(s: Status) {
  if (s === Status.Succes) {
    // 🔴 属性不存在，立刻报错
  }
}
用枚举管理状态码、方向、权限等「有限集合」， 杜绝字符串拼写错误，IDE 还能自动补全枚举项。
⑤ Class 访问修饰符 TS 对 Class 的增强
JavaScript
class Person {
  constructor(name, age) {
    this.name = name;
    this.age  = age;
    this._secret = "xyz";
    // _ 前缀只是"约定"私有
  }
}

const p = new Person("小明", 20);
console.log(p._secret); // 依然能访问！
TypeScript
class Person {
  public  name: string;
  private age:  number;
  readonly id:  number;

  constructor(name: string, age: number) {
    this.name = name;
    this.age  = age;
    this.id   = Date.now();
  }
}
const p = new Person("小明", 20);
p.age = 30; // 🔴 private，不可访问
TS 为 Class 新增 public / private / protected / readonly 修饰符， 真正实现封装，防止外部代码随意修改内部属性。

什么时候用哪个？

JS 选 JavaScript

  • ⚡ 小型项目、快速原型
  • 📚 初学前端基础阶段
  • 🧪 简单的脚本和工具
  • ⏱ 快速验证想法
  • 👤 个人独立开发项目
  • 🎓 练习算法和基础逻辑

TS 选 TypeScript

  • 🏢 大型团队协作项目
  • 🔄 长期维护的产品
  • ⚛️ Vue 3 / React 正式项目
  • 🛡 需要高可靠性的业务代码
  • 📖 代码即文档，便于协作
  • 🤖 想要强力 IDE 智能提示

给你的学习路线建议
🥚

第一阶段：打好 JS 基础

先把 JavaScript 核心概念学扎实：变量、函数、数组、对象、ES6+ 语法（箭头函数、解构、Promise），以及 DOM 操作。

🌱

第二阶段：Vue 3 + JS 起步

用 JavaScript 跑通 Vue 3 的基础功能（响应式、组件、路由），不要一开始就引入 TS 增加复杂度。

🚀

第三阶段：切换到 TS

Vue 基础打牢后，在项目中用 <script setup lang="ts">，体验类型推断和智能提示的强大。TS 就是加了类型的 JS，迁移很自然。

🎯

记住核心原则

TS 是 JS 的超集，所有 JS 知识在 TS 里都适用。学好 JS → TS 是水到渠成，反过来学会很吃力。

⚡

快速上手 TS 的技巧

把现有 .js 文件改成 .ts，让编译器告诉你哪里有问题。逐个修复类型错误，这是最高效的 TS 入门方式。

📦

用 Vite 快速搭建环境

用 npm create vite@latest 选 Vue + TypeScript 模板，Vite 已配置好 TS 编译，无需手动配置，直接开写。

快速参考：TS 核心语法
TS 常用类型注解速查 
// ===== 基础类型 =====
let name:    string   = "黎俊杰";
let age:     number   = 20;
let isDone:  boolean  = true;
let nothing: null    = null;
let undef:   undefined;

// ===== 数组 =====
let nums: number[]      = [1, 2, 3];
let strs: Array<string> = ["a", "b"];

// ===== 联合类型（或）=====
let id: number | string = 42;  // 数字或字符串都行

// ===== 类型别名 =====
type Point = { x: number; y: number };

// ===== 函数 =====
const greet = (name: string): string => {
  return `你好 ${name}`;
};

// ===== Vue 3 组合式 API 中的 TS =====
import { ref, computed } from "vue";

const count = ref<number>(0);   // 明确泛型类型
const double = computed(() => count.value * 2);
